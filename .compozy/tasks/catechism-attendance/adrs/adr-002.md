# ADR-002: Stack Técnica — Next.js + Supabase + Vercel

## Status

Accepted

## Date

2026-05-04

## Context

Projeto greenfield sem código preexistente. Requisitos-chave: PWA com suporte offline, autenticação por e-mail+senha com controle de acesso por role, banco relacional com RLS, geração de relatórios PDF e Excel, deploy simples. Escala pequena (até 10 turmas, 150 alunos). Time pequeno, entrega rápida de MVP.

## Decision

Adotar **Next.js 15 (App Router)** para frontend e API Routes, **Supabase** para banco PostgreSQL, autenticação e Row Level Security, e **Vercel** para deploy. Bibliotecas complementares: `jsPDF` + `xlsx` para relatórios, `Dexie.js` para IndexedDB offline, `shadcn/ui` + Tailwind CSS para UI, `react-hook-form` + `zod` para formulários.

## Alternatives Considered

### Alternative 1: Next.js + Neon (PostgreSQL gerenciado) + Prisma

- **Description**: Next.js com Prisma ORM apontando para Neon Postgres, sem BaaS.
- **Pros**: Mais controle sobre o banco; Prisma tem DX excelente; Neon tem tier gratuito.
- **Cons**: Precisa implementar autenticação do zero (NextAuth ou similar); sem RLS nativo; mais configuração de infra.
- **Why rejected**: Supabase já entrega auth + RLS + banco integrados, reduzindo significativamente o tempo de setup para o MVP.

### Alternative 2: React Native (Expo) + Backend separado

- **Description**: App nativo para catequistas; painel web separado para coordenador; API Node.js/Express própria.
- **Pros**: UX nativa superior no mobile; melhor suporte offline nativo.
- **Cons**: Duas codebases (mobile + web); publicação em App Store/Play Store; distribuição mais complexa para catequistas. O PRD exige que funcione no navegador sem instalação por loja.
- **Why rejected**: Contradiz requisito do PRD de funcionar no navegador do celular sem app store.

## Consequences

### Positive

- Supabase entrega auth, RLS e banco em um único serviço gerenciado.
- Next.js + Vercel = deploy com um comando, preview automático por PR.
- RLS do Supabase garante isolamento catequista/coordenador no nível do banco.
- Stack conhecida, com documentação abundante e ecossistema maduro.

### Negative

- Dependência de dois serviços externos (Supabase + Vercel); outage de qualquer um afeta o sistema.
- Supabase tem limites no tier gratuito (500 MB storage, 50.000 MAU) — adequado para o volume atual mas a monitorar.

### Risks

- Supabase descontinuar tier gratuito: migrar para plano pago é simples. Custo estimado: USD 25/mês no tier Pro.

## Implementation Notes

- Usar `supabase-js` v2 no client e Server Components do Next.js.
- RLS deve ser configurado antes de qualquer dado de produção entrar no banco.
- Variáveis de ambiente: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY` (apenas no servidor).
